# Rapport de Monétisation NutriProfile

## Stratégie Complète pour Développeur Individuel au Maroc

**Date**: Décembre 2024 (Mise à jour: Décembre 2025)
**Version**: 2.0
**Contexte**: Développeur individuel basé au Maroc, sans structure d'entreprise

---

## Sommaire Exécutif

NutriProfile possède un potentiel de monétisation significatif grâce à ses fonctionnalités d'analyse IA des repas, de suivi nutritionnel personnalisé et de génération de recettes. Ce rapport présente une stratégie réaliste et progressive, adaptée à un budget limité, avec un objectif de rentabilité en 6-12 mois.

**Objectif**: Atteindre 500€-2000€/mois de revenus récurrents dans les 12 premiers mois.

## 3. Implémentation Technique des Paiements - MAROC

### 3.0 Contexte Spécifique Maroc

En tant que développeur individuel au Maroc, vous avez plusieurs contraintes:
- **Stripe n'est PAS disponible directement** au Maroc
- **PayPal** a des limitations pour les retraits vers les banques marocaines
- Vous devez rapatrier vos devises en MAD sous 90 jours (réglementation de l'Office des Changes)

**Bonne nouvelle**: Plusieurs solutions Merchant of Record (MoR) supportent le Maroc !

### 3.1 Solutions Recommandées par Priorité (MAROC)

#### Option A: Lemon Squeezy (RECOMMANDÉ - Maroc supporté)

**Statut Maroc**: ✅ **SUPPORTÉ** - Payouts bancaires disponibles

**Pourquoi c'est la meilleure option ?**
- Merchant of Record = gère TVA/taxes automatiquement
- **Maroc dans la liste des pays supportés** pour les virements bancaires
- Approbation rapide (<48h)
- Parfait pour les indie developers sans entreprise
- Pas besoin de créer une société
- Programme d'affiliation intégré

**Tarification**: 5% + 0.50$ par transaction

**Paiements**:
- Virements le 1er et 15 de chaque mois
- Délai de retenue: 13 jours après vente
- 1-5 jours pour recevoir sur compte bancaire marocain

**Intégration**:
```typescript
// Exemple d'intégration LemonSqueezy
import { createCheckout } from '@lemonsqueezy/lemonsqueezy.js';

const checkout = await createCheckout({
  storeId: 'YOUR_STORE_ID',
  variantId: 'PREMIUM_VARIANT_ID',
  checkoutData: {
    email: user.email,
    custom: {
      user_id: user.id
    }
  }
});
```

**Webhook Backend (Python/FastAPI)**:
```python
from fastapi import APIRouter, Request, HTTPException
import hmac
import hashlib

router = APIRouter()

@router.post("/webhooks/lemonsqueezy")
async def handle_webhook(request: Request):
    payload = await request.body()
    signature = request.headers.get("X-Signature")

    # Vérifier la signature
    secret = settings.LEMONSQUEEZY_WEBHOOK_SECRET
    expected = hmac.new(
        secret.encode(), payload, hashlib.sha256
    ).hexdigest()

    if not hmac.compare_digest(signature, expected):
        raise HTTPException(status_code=401)

    data = await request.json()
    event = data["meta"]["event_name"]

    if event == "subscription_created":
        user_id = data["meta"]["custom_data"]["user_id"]
        # Activer premium pour l'utilisateur
        await activate_premium(user_id)

    return {"status": "ok"}
```

#### Option B: Polar.sh (Alternative moderne - Maroc supporté)

**Statut Maroc**: ✅ **SUPPORTÉ** via Stripe Connect Express

**Pourquoi Polar ?**
- Plateforme open-source, orientée développeurs
- **Frais les plus bas**: 4% + 0.40$ par transaction
- Merchant of Record (gère tout)
- Supporte les individus (pas besoin d'entreprise)
- Dashboard moderne, API bien documentée

**Limitations**:
- Plus récent, moins de fonctionnalités que Lemon Squeezy
- Communauté plus petite

**Tarification**: 4% + 0.40$ (le moins cher du marché MoR)

**Intégration**:
```typescript
// Polar.sh Checkout
import { Polar } from '@polar-sh/sdk';

const polar = new Polar({ accessToken: process.env.POLAR_ACCESS_TOKEN });

const checkout = await polar.checkouts.create({
  productId: 'PREMIUM_PRODUCT_ID',
  successUrl: 'https://nutriprofile.app/success',
  customerEmail: user.email,
  metadata: { user_id: user.id }
});
```

## 6. PWA vs Application Native

### 6.1 Situation Actuelle

NutriProfile est actuellement une **PWA (Progressive Web App)**. Voici l'analyse:

#### Avantages PWA (Situation Actuelle)

| Avantage | Impact |
|----------|--------|
| Pas de commission App Store (15-30%) | +15-30% revenus |
| Déploiement instantané | Itérations rapides |
| Un seul codebase | Moins de maintenance |
| Accessible via URL | Partage facile |

#### Inconvénients PWA

| Inconvénient | Impact |
|--------------|--------|
| Pas de visibilité App Store | Moins de découvrabilité |
| Push notifications limitées (iOS) | Engagement réduit |
| Perception "moins sérieux" | Confiance utilisateur |

### 6.2 Recommandation

**Court terme (0-12 mois)**: Rester en PWA
- Focus sur le produit et premiers revenus
- Économie des frais Apple/Google
- Itération rapide

**Moyen terme (12-24 mois)**: Considérer app native si:
- Revenus > 5,000€/mois
- Base utilisateurs > 50,000
- Besoin de fonctionnalités natives (HealthKit, etc.)
